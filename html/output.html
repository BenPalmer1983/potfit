Standard Output

potfit prints information on the minimization progress to stdout. Additionally all input and output files used are also written to standard output. The output will also be written to standard output unless the output_prefix parameter is set.
After optimizing the potential, the local densities (eam only) are printed, followed by the terms in the sum of squares. The latter consist of a number identifying either atom or configuration, then the contribution to the sum of squares, the value calculated from the potential, the reference value and finally the relative deviation of calculated and reference value.

Potfit Output Files

Final potential

potfit naturally writes the potential achieved in minimization to a file using the same format as for the input potential. The name of this file is specified by the endpot parameter.

Temporary Potential File

Every now and then (after restart of Powell algorithm or step adjustment of simulated annealing) an intermediate potential is written to tempfile.

IMD Potential File

Prefix of files to which the IMD potential data are written. The suffixes used are self-explanatory, the number of steps in each potential is given by the imdsteps parameter

Plotting files

Two files are printed to use with gnuplot: The plotpointfile contains the sampling points in gnuplot-readable form, whereas the plotfile uses spline-interpolated points to draw continuous curves of the potential functions. As potfit uses spline interpolation internally, the plotted functions are an accurate representation of the potentials used.

Output files

If the output_prefix parameter is set in the configuration file, potfit will write several files, else all information will be printed to standard output.
In detail the files written are:

.force contains the calculated and reference forces
.energy contains the calculated and reference energies
.stress contains the calculated and reference stresses
.punish contains the different punishments
.rho_loc local electron density
