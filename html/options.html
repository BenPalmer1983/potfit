Compilation Options

In addition to selecting parallelization method (e.g., MPI), various options can be compiled into potfit.

We give here a list of all possible options, and describe what their purpose is and how they are used.

Possible Options in the Compilation Target

acml apot debug dist eam evo
fweight newscale noresc pair parab stress
acml

Use AMD Core Math Library instead of Intel Math Kernel Library. Also requires manual changes to the Makefile.

apot

Enable analytic potentials.

debug

Compile with debug flags. Also provides more information (minimal distances matrix, minimization output). Will produce lots of output.

dist

Information on how often each segment (between two sampling points) of the various functions is used in force calculation is printed to the file given by the parameter distfile. Useful to check whether sampling points have a sensible density and/or distribution. This option is incompatible with mpi parallelization.

eam

Use Embedded Atom Method potentials.

evo

Use evolutionary algorithm instead of simmulated annealing.

fweight

Use modified weights for the forces. Each force component F_j is weighted by the factor ( || F || + FORCE_EPS )^(-1). This increases the importance of numerically small forces. The compile time parameter FORCE_EPS prevents the overweighting of very tiny forces.

newscale

An additional set of potential plotting files and imd potential files is written to disk. The file names are formed by appending _new to the original file names. Those potentials are rescaled to a more common variety of the gauge degree of freedom with eam potentials: The gradient of the embedding function is zero at the global mean electron density (averaged over all atoms in all configurations). This option is incompatible with mpi parallelization.

noresc

Suppresses rescaling of the potential. Useful if close to the minimum and a rescaling would ruin the minimization.

pair

Use pair potentials.

parab

Use parabolic interpolation with the embedding functions. Will only work, if embedding functions are given at exactly three sampling points. Parabolas allow for extrapolation, which can be useful if you are far from the minimum.

stress

The option stress enables the computation of the stress or pressure tensor and its usage in minimization. Configurations then need to include stress tensor information.

Deprecated Options in the Compilation Target

limit

This former option is now mandatory! Use limiting constraints to restrain gauge invariance of EAM potentials. An additional term is added to the sum of squares, if one of "electron densities" within one configuration lies outside the domain of the embedding functions.
